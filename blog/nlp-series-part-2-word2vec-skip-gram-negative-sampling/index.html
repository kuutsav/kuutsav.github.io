Posted December 22, 2021 · 10 min read

Continuing with our previous post, we explore Word2Vec skip-gram model with negative sampling.

Most of the steps are similar to our notebook on Word2Vec without negative sampling. The key changes happen with the dataset where we provide a list of indices for negative sample for each forward pass as well as the loss function.

In this notebook, we will train a Word2Vec skip-gram model with negative sampling from scratch on some text and inspect the trained embeddings at the end.

The architecture we define later will enable us to learn a distributed representation for each word/token in our corpus. The key idea being, we can represent a work by adding contextual information to it. Context here is the neighbouring words around a word/token of interest to us.

from collections import Counter

The whole process can be broken down into the following key steps:

  1. Fetch and process text using TextProcessor.
  2. fetch_text() fetches the texts directly from urls containing our text data.
  3. process_text() applies basic pre-procesing steps to the text and creats tokens using white space tokenization.
  4. prepare_vocab() creates token to index mapping and vice versa.
  5. Create torch Dataset and Dataloader where each index in the dataset will be a tuple, (center_word_ix, context_word_ix); i.e. pairs of indices of center word and context based on a pre defined window size.
  6. Create our SkipGramModel model based on the torch.nn.Module.
  7. Define our loss function, optimizer and scheduler.
  8. Train the model for a certain number of epochs.

Finally, we will create a similarity matrix(cosine similarity) using the trained embeddings and explore similar embeddings for some word query.

class TextProcessor:
    def __init__(self, urls: List[str]):
        self.urls = urls

    def fetch_text(self):
        self.text = ""
        for i, url in enumerate(self.urls):
            r = requests.get(url)
            r.status_code
            self.text += "\n\n\n" + r.text
            print(f"Fetched {i+1}/{len(self.urls)} urls ...")

    def process_text(self):
        print("Processing text ...")
        sentences = re.split("\n{2,}", self.text)
        print(f" > {len(sentences)} sentences")
        self.clean_sentences = []
        for txt in sentences:
            txt = txt.strip().lower()
            txt = txt.replace("\n", "")
            txt = " ".join(re.findall("\w+", txt))
            if txt:
                self.clean_sentences.append(txt)

        # Tokens and token counter will be used later to create a vocabulary for
        # us which will be used to map words/tokens to indices and vice versa, to
        # create training data and recreate the texts from predictions respectively.
        print(f" > {len(sentences)} filtered sentences")
        tokens = " ".join(self.clean_sentences).split(" ")
        print(f" > {len(tokens)} tokens")
        self.token_counter = Counter(tokens)
        print(f" > {len(self.token_counter)} unique tokens")

    def prepare_vocab(self, min_count: int):
        # min_count is the minimum number of times a token should appear in the
        # text to be considered in the vocabulary else they are assigned to a
        # default index which is equal to `len(vocab)`.
        self.w2ix = {}
        for i, (token, count) in enumerate(
            self.token_counter.most_common(len(self.token_counter))
        ):
          if count < min_count:
              break
          else:
              self.w2ix[token] = i

        self.ix2w = {ix: w for w, ix in self.w2ix.items()}

        # assign default index to rest of the tokens
        n = len(self.w2ix)
        for token, _ in self.token_counter.most_common(len(self.token_counter)):
            if token not in self.w2ix:
              self.w2ix[token] = n
        self.ix2w[n] = "<unk>"
        self.ixs = list(self.ix2w.keys())

        self.vocab_size = n + 1
        print(f"Vocab size: {self.vocab_size}")

Step 2. Prepare the torch Dataset and Dataloader #

The Dataloader will enable us to send data in batches during the forward pass of the model training.

N_NEG_SAMPLES = 20

class HarryPotterDataset(Dataset):
    def __init__(self, text_processor: TextProcessor,  window_size: int=3):
        self.data = []
        for txt in text_processor.clean_sentences:
            splits = txt.split(" ")
            for i in range(window_size, len(splits) -1):
                center_word = splits[i]
                window_words = splits[i-window_size:i]+splits[i+1:i+window_size+1]
                for context in window_words:
                    # Each data point under self.data will be a tuple with index 0
                    # containing the index(w2ix) for center_word and
                    # index 1 containing the index(w2ix) for context_word and
                    # index 2 containing negative samples
                    self.data.append((
                        text_processor.w2ix[center_word], text_processor.w2ix[context],
                        sample(text_processor.ixs, N_NEG_SAMPLES)
                    ))

    def __getitem__(self, ix: int):
        return self.data[ix]

    def __len__(self):
        return len(self.data)

The Dataset essentially contains indexes of pairs of center word and a context word.

def collate_fn(collate_fn_input: List[Tuple]):
    return (
        torch.LongTensor([i[0] for i in collate_fn_input]), # center word
        torch.LongTensor([i[1] for i in collate_fn_input]), # context word
        torch.LongTensor([i[2] for i in collate_fn_input])  # negative samples
    )

dataloader = DataLoader(dataset, batch_size=64, collate_fn=collate_fn, shuffle=True)

Step 3. Create the Word2Vec model using torch.nn.Module. #

class SkipGramModel(nn.Module):
    def __init__(self, vocab_size: int, embedding_size: int=50):
        super().__init__()
        self.u_embeddings = nn.Embedding(vocab_size, embedding_size)
        self.v_embeddings = nn.Embedding(vocab_size, embedding_size)

    def forward(self, pos_u, pos_v, neg_v):
        losses = []
        emb_u = self.u_embeddings(pos_u)
        emb_v = self.v_embeddings(pos_v)
        score = torch.mul(emb_u, emb_v).squeeze()
        score = torch.sum(score, dim=1)
        score = nn.functional.logsigmoid(score)
        losses.append(sum(score))
        neg_emb_v = self.v_embeddings(neg_v)
        neg_score = torch.bmm(neg_emb_v, emb_u.unsqueeze(2)).squeeze()
        neg_score = torch.sum(neg_score, dim=1)
        neg_score = nn.functional.logsigmoid(-1 * neg_score)
        losses.append(sum(neg_score))
        return -1 * sum(losses)

Embedding is nothing but “A simple lookup table that stores embeddings of a fixed dictionary and size” - pytorch documentation.

The modified loss function for negative sampling is:

Source: https://lena-voita.github.io/nlp_course/word_embeddings.html

In each forward pass we:

  1. Pass indices of center word as input param pos_u.
  2. These indices are looked up in the self.u_embeddings table.
  3. pos_v is the index for context word and self.v_embeddings is the associated embedding table.
  4. score is the first part of the loss function, i.e. the component we need to increase(similarity between the center and context vectors)
  5. neg_score is the second part of the loss function, i.e. the component we need to decrease(similarity between the center embedding and negative embeddings)

A trained model should output high values for the context indices for a given center word.

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

Step 5. Train the model #

epoch_losses = []
batch_losses = []
for epoch in range(N_EPOCHS):
    losses = []
    for i, data in enumerate(dataloader):
        inputs = data[0].to(device)
        outputs = data[1].to(device)
        neg_samples = data[2].to(device)
        loss = model(inputs, outputs, neg_samples)
        losses.append(loss.item())
        loss.backward()
        optimizer.step()
        optimizer.zero_grad()
        if i % STEP_TO_LOG == 0:
            print(f"epoch {epoch + 1}; steps={i:>5}/{len(dataloader)}; loss={loss:<.5f}")
    scheduler.step()
    batch_losses += losses
    epoch_losses.append(np.mean(losses))
    print(f"epoch {epoch + 1} ; lr={scheduler._last_lr[0]}; loss={loss:<.5f}\n")

Step 6. Create the similarity matrix #

similarity = cosine_similarity(weights, weights)

def get_similar_words(input_word: str, n: int):
    if input_word not in text_processor.w2ix:
        print("word not in vocab")
    else:
        input_word_ix = text_processor.w2ix[input_word]
        similarity_vector = similarity[input_word_ix]
        most_similar_ixs = np.argsort(similarity_vector)[-n:][::-1]
        most_similar_words = [(text_processor.ix2w[ix], similarity_vector[ix])
                              for ix in most_similar_ixs]
        print(input_word)
        for word, score in most_similar_words:
            print(f" > {word}, score={score: .2f}")
