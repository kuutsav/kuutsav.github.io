In this notebook, we will train a LSTM model to generate text.

Such models are called “Language Models”, trained to recurrently produce text. We can also use such models to assign probabilites to pieces of text.

We will be training a character level LSTM but it can be trained at word level as well(vocabulary size will be much higher in case of words).

Within each forward pass, the model will receive indices for a sequence and will be tasked to predict the next character at each time step.

For example:

Input : The boy who lived

Output : he boy who lived.

Step 2. Create the LSTM model #

class RnnLanguageModel(nn.Module):
  def __init__(self, vocab_size, embedding_size, hidden_size, num_layers):
    super().__init__()
    self.embedding = nn.Embedding(vocab_size, embedding_size)
    self.rnn = nn.LSTM(embedding_size, hidden_size, num_layers=num_layers)
    self.linear = nn.Linear(hidden_size, vocab_size)

  def forward(self, input_seq, hidden):
    embedding = self.embedding(input_seq)
    output, hidden = self.rnn(embedding, hidden)
    output = self.linear(output)
    return output, (hidden[0].detach(), hidden[1].detach())

model = RnnLanguageModel(vocab_size=len(chars), embedding_size=50,
                         hidden_size=256, num_layers=2).to(device)

Step 3. Define our loss function, optimizer and scheduler #

criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.002)
scheduler = torch.optim.lr_scheduler.StepLR(optimizer, step_size=1, gamma=0.5)

In each forward pass, we will be sending in a sequence of indices whose length will be between MIN_SEQ_LEN and MAX_SEQ_LEN.

MIN_SEQ_LEN = 10
MAX_SEQ_LEN = 100
N = len(data_ixs)

We will also define a helper function to help us generate text from an input prompt.

This will also help us visually gauge the training progress by generating pieces of text after a certain number of training steps has finished.

def generate(txt, hidden, n=100):
  output_txt = txt
  for i in range(n):
    txt2ix = [c2ix[c] for c in txt]
    input_seq = torch.unsqueeze(torch.LongTensor(txt2ix), dim=1)
    pred, hidden = model(input_seq, hidden)
    pred = nn.functional.softmax(pred.squeeze(), dim=0)
    # Here, we are sampling instead of taking an argmax to add diversity
    # to the generated text.
    # Using argmax will lead to the model generating repetitive patterns.
    # We will cover this topic in much more detail when we revist
    # language models under the transformers architecture.
    dist = torch.distributions.Categorical(pred)
    index = dist.sample()
    pred_char = ix2c[index.item()]
    output_txt += pred_char
    txt = pred_char
  return output_txt

Step 4. Train the model #

n = 0
losses = []
hidden = None
while True:
  start_ix = random.randint(0, N-MAX_SEQ_LEN-1)
  seq_len = random.randint(MIN_SEQ_LEN, MAX_SEQ_LEN)
  input_seq = data_ixs[start_ix: start_ix+seq_len]
  target_seq = data_ixs[start_ix+1: start_ix+1+seq_len]
  pred, hidden = model(input_seq, hidden)
  loss = criterion(pred.squeeze(), target_seq.squeeze())
  losses.append(loss.item())
  loss.backward()
  nn.utils.clip_grad_norm_(model.parameters(), 5)
  optimizer.step()
  optimizer.zero_grad()
  n += 1
  if n > 50000:
    break
  if n % 1000 == 0:
    print(f"step {n:<5}; mean_loss={np.mean(losses):.5f}")
    print(f"text generated with prompt 'h' -> {repr(generate('h', hidden))}")
    losses = []
  if n % 10000 == 0:
    scheduler.step()
    print(f"updating lr -> {scheduler._last_lr[0]}\n")

LSTMs can easily be modified to accomodate other architectures. For example:

  • We can send in a sequence and use softmax on the output to predict the next charater, use the actual next character from data as target, compute loss and back propogate.
  • Combine all the embeddings from the hidden layer and apply softmax to predict the next character. Combination could be a mean or index wise max, etc.
